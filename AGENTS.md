# 项目规范 —— AI 实战技能知识库

## 项目概述
- 本地 Markdown 知识库
- 技术栈：纯 Markdown + Git 版本控制
- 目标：系统整理 AI 开发、UI/UX 设计、知识管理领域的核心资源

## 工作流（强制）
1. **理解**：复述需求，指出模糊点，提出澄清问题
2. **规划**：输出实施步骤（不直接写代码）
3. **执行**：按步骤执行，每步完成后验证
4. **审查**：检查是否引入无关修改，diff 是否最小化

## 编码/写作规范
- Simplicity First：不添加未请求的内容
- Surgical Changes：只改必要的文件
- Goal-Driven：定义成功标准，验证后交付

## 常用命令
- 查看状态：`git status`
- 提交更改：`git add -A && git commit -m "描述"`
- 推送到远程：`git push`
