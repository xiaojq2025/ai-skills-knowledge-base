# AnySearch（垂直搜索）：AI 时代的搜索基础设施

> 统一 API 聚合金融、法律、学术等垂直数据，返回结构化 Markdown，填补通用搜索 80% 盲区。专为 AI Agent 打造的下一代搜索基础设施。

---

## 🎯 核心定位

**AnySearch** 不做"又一个 AI 搜索产品"，而是做 **AI 时代的搜索基础设施**。核心判断：

> "互联网上约 80% 的高价值信息——行业数据库、实时金融终端、代码仓库、学术平台、企业工商系统，并不在传统搜索引擎的可达范围内。"

---

## 🏗️ 三大设计要素

### 1. 自建索引与多源聚合
穿透"表层互联网"，覆盖 23+ 个专业垂直领域：

| 领域 | 数据来源 |
|------|---------|
| **金融** | 美股/港股/A股、实时估值、融资节奏 |
| **法律** | 判例数据库、法规条文 |
| **学术** | 论文库、研究成果 |
| **安全** | CVE 漏洞库、威胁情报 |
| **代码** | GitHub、代码仓库深度索引 |
| **商业** | 企业工商、知识产权 |
| **能源/环境** | 行业数据库 |

### 2. 面向 Agent 的结构化输出
- **智能意图路由**：自动识别查询意图，路由到最合适的数据源
- **返回结构化 Markdown**：Agent 无需处理 HTML、过滤广告噪音
- **直接用于推理和执行**：拿到手的就是干净数据

### 3. 统一接口极简部署
- **三种接入方式**：Skill / MCP / API
- **一个统一 API** 覆盖多个专业领域
- **个人开发者免费**（每日 1000 次调用额度）
- **部署流程**：复制命令 → 几秒完成

---

## 🔧 MCP Server 核心工具

| 工具 | 功能 | 适用场景 |
|------|------|---------|
| `search` | 通用搜索 + 23 个垂直领域搜索 | 金融、学术、安全、法律等 |
| `list_domains` | 查询垂直领域目录 | Agent 首次使用时发现可用子域 |
| `batch_search` | 并行执行 2-5 个独立查询 | 一次请求解决多个意图 |
| `extract` | 提取 URL 全文内容（返回 Markdown） | 需要比摘要更深的内容 |
