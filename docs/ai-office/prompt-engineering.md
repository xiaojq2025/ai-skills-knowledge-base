# 提示工程实战：办公场景 Prompt 设计方法论

> 不是让你背 Prompt，而是学会"怎么跟 AI 说清楚你的需求"。适用于 Kimi、Claude、ChatGPT 等所有大模型。

---

## 🎯 办公场景 Prompt 的黄金公式

```
角色 + 任务 + 背景 + 要求 + 格式
```

| 要素 | 说明 | 示例 |
|------|------|------|
| **角色** | 你希望 AI 扮演谁 | "你是一位资深行政经理" |
| **任务** | 具体要做什么 | "帮我写一封会议延期通知" |
| **背景** | 相关上下文 | "原定下周三的季度评审会，因总监出差需延期" |
| **要求** | 限制条件 | "语气委婉，给出两个备选时间，200字以内" |
| **格式** | 输出形式 | "邮件格式，包含主题行" |

**完整示例**：
> 你是一位资深行政经理。帮我写一封会议延期通知。原定下周三的季度评审会，因总监出差需延期。语气委婉，给出两个备选时间（下周五上午和下周一上午），200字以内，邮件格式，包含主题行。

## 🏗️ 进阶技巧：CO-STAR 框架

来自新加坡政府科技局的官方 Prompt 工程框架，特别适合办公场景：

```
C - Context（上下文）：背景信息
O - Objective（目标）：要达成什么
S - Style（风格）：语气、写作风格
T - Tone（语调）：正式、轻松、紧迫...
A - Audience（受众）：给谁看的
R - Response（响应格式）：输出形式
```

**示例：写产品发布通知**

```
【Context】我们公司新产品"智能考勤系统"将于下月1日正式上线
【Objective】写一封内部通知，让全体员工了解新系统并配合切换
【Style】清晰简洁，要点明确
【Tone】正式但不生硬，带有鼓励性
【Audience】公司全体员工，包括技术和非技术人员
【Response】邮件格式，分"背景、变化、需要做什么、支持渠道"四部分
```

## ⚡ 办公人员必学的 5 个 Prompt 技巧

### 技巧 1：让 AI 先问问题（反向提问）
当你需求模糊时，让 AI 帮你理清：
> "我想写一份项目提案，但还没想清楚。你先问我 5 个关键问题，帮我理清思路，我们再一起写。"

### 技巧 2：给示例（Few-Shot）
给 AI 一个你想要的格式示例：
> "请按以下格式写周报：
> 【本周完成】
> - 事项1：完成情况
> - 事项2：完成情况
> 【下周计划】
> - 事项1：预期目标
> 【需要支持】
> - ..."

### 技巧 3：分步执行（Chain of Thought）
复杂任务不要让 AI 一次做完：
> "我们分三步写这个方案：
> 第一步：你先列出这个方案应该包含哪些章节
> 第二步：我确认后，你写第一章的详细内容
> 第三步：逐章推进"

### 技巧 4：设定边界（Negative Prompt）
告诉 AI 不要做什么：
> "写一份活动通知。注意：不要出现'敬请期待'这类空泛词汇，不要加任何表情符号，不要假设领导会出席。"

### 技巧 5：迭代优化
第一轮输出永远不是最终版：
> "这个版本太长了，缩短到 300 字"  
> "语气太正式了，我们内部用的，轻松一点"  
> "第三点不够清楚，加一个简单的例子"

## ❌ 办公场景常见 Prompt 错误

| 错误 | 为什么不好 | 改进 |
|------|-----------|------|
| "写得好一点" | AI 不知道"好"的标准 | "增加具体数据支撑，减少形容词" |
| "专业一点" | 太模糊 | "使用行业术语，符合商务邮件规范" |
| "尽快" | 无法量化 | "本周五下班前需要发出" |
| "你懂的" | AI 真不懂 | 把默认 AI 不知道的事情都说清楚 |
| 一次要太多 | 输出质量差 | 拆成 2-3 轮对话 |
